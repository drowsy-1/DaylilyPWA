## IMPLEMENTATION RECOMMENDATIONS

### Core Fields (Always Track)
These should be in your main variety table:
- All Basic Identifiers (7)
- Key measurements: scape_height, bloom_size, branch_count, bud_count (4)
- Key classifications: form_type, base_color, ploidy, foliage_type (4)
- Fertility: pollen_fertility, pod_fertility (2)

**Core Total: ~20 fields**

### Standard Observations
These should be tracked per observation event:
- Scape & bloom measurements (can change year to year)
- Color performance (fading, intensity)
- Weather tolerance observations
- Vigor ratings
- Disease/pest issues

### Advanced/Optional Traits
Track these as custom traits based on your breeding goals:
- Detailed color features (edge, eye, watermark)
- Teeth measurements
- Fragrance details
- Detailed environmental responses

### Derived/Calculated Fields
These don't need manual entry:
- Genetic metrics (from pedigree analysis)
- Breeding values (from Bayesian models)
- Performance scores (from aggregating observations)

## NOTES ON MEASUREMENT STANDARDS

Some fields benefit from standardized measurement protocols:

**Measurements requiring tools:**
- bloom_size: measure widest point, same time of day
- scape_height: ground to top of tallest bud
- petal_width: widest point of typical petal

**Ratings requiring calibration:**
- Establish reference plants for 1, 5, and 10 ratings
- Use consistent raters when possible
- Photo reference library helps

**Color:**
- Always note lighting conditions
- Time of day affects color
- Consider using color cards for consistency
